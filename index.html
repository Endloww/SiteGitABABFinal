Issues

Le menu "Issues" dans GitHub est un outil de gestion de projet qui permet aux utilisateurs de signaler des problèmes, des bogues ou des demandes de fonctionnalités pour un projet. Il permet également aux membres de l'équipe du projet de suivre ces problèmes, de les trier par priorité, de les affecter à des membres spécifiques de l'équipe pour les résoudre, et de tenir les utilisateurs informés de l'état des problèmes.

Pull requests

Le menu "Pull Requests" dans GitHub est un outil de collaboration de code qui permet aux développeurs de proposer des modifications de code et de les soumettre pour examen et fusion dans le code principal du projet. Les pull requests permettent aux membres de l'équipe de discuter des modifications de code et de les réviser avant qu'elles ne soient fusionnées, ce qui permet de maintenir la qualité du code et d'éviter les conflits et les erreurs.

Actions

Le menu "Actions" dans GitHub est un outil d'automatisation des tâches qui permet aux utilisateurs de créer et d'exécuter des workflows personnalisés pour leur projet. Les workflows d'actions peuvent inclure des tâches telles que la compilation de code, les tests de code, la génération de documentation et le déploiement de code sur des serveurs ou des services cloud. Les workflows peuvent être configurés pour s'exécuter automatiquement en réponse à des événements tels que des pull requests ou des modifications de code.

Visual Studio Code


  • Tout d'abord, il est important de noter que Visual Studio prend en charge l'intégration avec GitHub. Cela signifie que les développeurs peuvent utiliser Visual Studio pour travailler sur leur code et le pousser directement sur GitHub.

    De plus, Visual Studio offre une intégration transparente avec la fonctionnalité de contrôle de version Git de GitHub, ce qui facilite la gestion des versions de code.

  • Visual Studio est un IDE puissant qui offre une multitude de fonctionnalités pour améliorer la productivité des développeurs. Par exemple, il prend en charge les langages de programmation les plus courants, tels que C++, C#, Python, Java, etc.

    Il dispose également de nombreuses fonctionnalités de débogage et de refactoring, ce qui facilite la correction des erreurs et la maintenance du code.



  • D'autre part, GitHub est une plate-forme de gestion de code source qui permet aux développeurs de stocker et de gérer leur code. GitHub prend en charge Git, un système de contrôle de version qui facilite la gestion des versions de code. GitHub offre également une multitude de fonctionnalités pour la collaboration, telles que des outils de suivi des problèmes, des commentaires de code et des revues de code.

    En combinant les fonctionnalités de Visual Studio et de GitHub, les développeurs peuvent bénéficier de la productivité de Visual Studio et de la collaboration de GitHub. Visual Studio facilite le développement de logiciels, tandis que GitHub facilite la gestion de code source et la collaboration entre les membres de l'équipe.

Création Token

  • Se rendre sur son profil GitHub 1
  • Aller dans "Settings" 2
  • Puis "Developer settings" 3
  • Cliquez sur "Personal access tokens" 4
  • Puis "Generate new token" 5
  • Nommez votre Token, puis sélectionnez les autorisations 6
  • Cliquez sur "Generate token" 7
  • Copiez le jeton généré et conservez-le en sécurité 8

Un token Github est donc un jeton d'authentification permettant d'effectuer des opérations avec votre compte Github sans avoir à saisir vos informations d'identifications à chaque fois. Les étapes à gauche sont les étapes à suivre pour en créer un.

Une fois que vous avez généré un jeton Github, vous pouvez l'utiliser pour effectuer des opérations push/pull avec un référentiel Github. Pour cela, vous devez inclure votre jeton dans les URL de votre référentiel Github lors de l'exécution de commandes push/pull. Par exemple, si votre jeton est "abcdefg12345", vous pouvez utiliser la commande suivante pour pousser vers votre référentiel: 

git push https://github.com/nom_utilisateur/referentiel.git?access_token=abcdefg12345

    Branches

    Créer une nouvelle branche et basculer dessus: 

    "git checkout -b <nom de la branche>"


    Passer d'une branche à l'autre: 

    "git checkout <nom de la branche>"


    Afficher la liste des branches du dépôt et celle dans laquelle vous êtes: 

    "branche git"


    Supprimer la branche de fonctionnalités: 

    "git branch -d <nom de la branche>"


    Mettre une branche vers le dépôt distant 

    "git push origin <nom de la branche>"


    Mettre toutes les branche vers le dépôt distant: 

    "git push --all origin <nom de la branche>"


    Supprimer une branche de votre dépôt: 

    "git push origin : <nom de la branche>"

    MAJ Dépôt

    Récuperer et fusionner les modifications sur le serveur distant du dépôt: 

    "git pull"


    Fusionner une branche différente dans votre branche: 

    "git merge <nom de la branche>"


    Afficher les conflits de fusion: 

    "git diff"


    Afficher les conflits avec le fichier de base: 

    "git diff --base <nom du fichier> "


    Visualiser les modifications avant la fusion: 

    "git diff <branche source> <branche cible> "


    Marquer le fichier modifié après résolution des conflits: 

    "git add <nom du fichier> "

    Dépôt

    Créer un nouveau dépôt local: 

    "git init"


    Créer une copie d'un dépôt: 

    "git clone /chemin/vers/dépôt"


    Créer une copie d'un dépôt: 

    "git clone /chemin/vers/dépôt"


    Pour un serveur distant utilisez: 

    "git clone username@host:/chemin/vers/dépôt"


    Ajouter un ou plusieurs fichier(s) à l'index 

    "git add <nom du fichier>" ou "git add *"


    Valider les modifications (pas encore dans le dépôt distant): 

    "git commit -m "Message de validation"


    Valider tout les fichiers ajoutés avec "git add": 

    "git commit -a"


    Répertorier les fichiers modifiés ceux en cours de validation: 

    "statut git"

    Annuler

    Remplacer les changements dans l'arbre de travail par le contenu de "head": 

    "git checkout -- <nom du fichier> "


    Supprimer tout les changements et récuperer le dernier historique du serveur: 

     1. "git fetch origin"

    2. "git reset --hard origin/master"
